# MLB Schedule Optimization — DASC 2026

This repository contains the work for our **Mixed-Integer Linear Programming (MILP)** project to optimize the Major League Baseball (MLB) schedule. The primary goal is to minimize total league-wide travel distance across the 162-game season, subject to real-world scheduling constraints such as mandatory series lengths, matchup quotas, and consecutive home/away limits.

## EDA Notebook (`Optimization_EDA.ipynb`)

### Part 1 — Full 30-Team Analysis (Tasks 1–4)

| Task | Output | MILP Role |
|------|--------|-----------|
| **Task 1** — Sets & Mappings | 30 teams, 30 stadiums, home-park map `H_a` | Index sets and home-location constraints |
| **Task 2** — Matchup Matrix | 30×30 integer matrix `M(a,j)` | Game-count constraints: `Σ G(a,j,d) = M(a,j)` |
| **Task 3** — Distance Matrix | 30×30 Haversine distance matrix `Dist(b,c)` | Objective function coefficients |
| **Task 4** — Baseline Travel | Per-team and league-wide historical miles | Upper bound / benchmark for the optimizer |

### Part 2 — AL East Division Focus (Steps 1–7)

| Step | What It Does |
|------|-------------|
| **Step 1** | Justifies the scope reduction (~200× fewer variables) |
| **Step 2** | Filters the full schedule to AL East intra-division games |
| **Step 3** | Extracts the 5×5 matchup sub-matrix and cross-validates |
| **Step 4** | Extracts the 5×5 distance sub-matrix |
| **Step 5** | Computes AL East division-only baseline travel |
| **Step 6** | Exports `al_east_schedule.csv` and `al_east_distance_matrix.csv` |
| **Step 7** | Reconciliation table — proves every model input traces to this EDA |

## Optimization Model (`OP.ipynb`)

A Pyomo MILP model scoped to the 5-team AL East division. Uses the CSV files exported by the EDA as input.

**Key result (AL East):**

| Metric | Value |
|--------|-------|
| Actual 2025 travel | 35,774.9 miles |
| Optimized travel | 12,745.2 miles |
| Savings | **23,029.8 miles (64.4%)** |

## Getting Started

**For the EDA:** Open `Optimization_EDA.ipynb` in Jupyter/VS Code. Run all cells top-to-bottom. Requires `pandas`, `numpy`, `matplotlib`, `seaborn`.

**For the MILP model:** Open `OP.ipynb` in Jupyter/VS Code. Requires `pyomo` and `highspy`. The CSV inputs are generated by the EDA notebook.
